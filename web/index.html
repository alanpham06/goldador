Labrador Governance

Goldador is the shared source of truth for Labrador membership, team access, and leadership permissions. Use this page to understand what access each role receives and learn how to register yourself as a member or add yourself to a team.

Role Permissions

Member

  • GitHub organization member access.
  • ScottyLabs Google Group member access.
  • ScottyLabs Google Drive contributor access.

Team Member

  • Membership in the team’s GitHub team.
  • Write access to the team’s GitHub repositories.
  • Membership in the team’s Slack channel.
  • Keycloak project team member access.
  • Read/list access to local project secrets in OpenBao.

Team Lead

  • Admin access to the team’s GitHub repositories.
  • Codeowners review ownership for the team’s Goldador team file.
  • Keycloak project team admin access.
  • Create, read, update, delete, list, and sudo access to the team’s project secrets in OpenBao.

Leadership Member

  • Keycloak read-only access.
  • Admin access to secrets in OpenBao.

Leadership Admins

  • GitHub organization admin access.
  • ScottyLabs Google Group manager access.
  • ScottyLabs Google Drive content manager access.
  • Keycloak admin access.
  • Admin access in OpenBao.
  • Default Codeowners review ownership for Goldador and the teams/ directory.

Register or Update Access

Step 1

Link Your Accounts

  • Create a Keycloak account .
  • Link your GitHub account to Keycloak.
  • Link your Slack account to Keycloak.
Step 2

Set Up Your Fork

  • Fork the Goldador repository, then clone your fork (GitHub cloning guide).
  • Install the recommended VS Code extensions to auto-format files (see CONTRIBUTING.md ).
  • Open the repo in your editor and confirm formatters apply on save.
Step 3

Add Yourself

  • Create your member file members/<github-username>.toml (member schema).
  • Add yourself to your team in teams/<slug>.toml; team leads create or update the team file (team schema).
  • One change per PR: add yourself as a member, or join/create one team — not both in the same PR.
Step 4

Open a PR for Review

  • Open a PR from your fork to main and submit it from your own GitHub account.
  • CODEOWNERS auto-requests reviewers; CI must pass before merge (see validator README for what’s checked).
  • Reviewers squash-merge approved PRs (full process in CONTRIBUTING.md ).

Ready to make a change?

Open the Goldador repository, edit the relevant TOML file, and submit a pull request for review.
